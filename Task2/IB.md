# Проверочный лист безопасности бизнес-систем PropDevelopment

## Обоснование выбора разделов:

### 1. Управление доступом (Высокий приоритет)

**Обоснование:**

- Серьезные проблемы с контролем доступа, приводящие к утечкам данных
- Несколько точек регистрации клиентов, отсутствие единого реестра
- Проблемы с API партнеров, позволяющие доступ к чужим данным
- Высокие репутационные и финансовые риски

### 2. Безопасность данных (Высокий приоритет)

**Обоснование:**

- Жалобы клиентов на отображение чужих данных в личных кабинетах
- Планы по внедрению BI/ML требуют надежной защиты данных
- Отсутствие Data Governance и классификации данных

### 3. Защита инфраструктуры (Средний приоритет)

**Обоснование:**

- Гибридная инфраструктура требует дополнительных усилий по безопасности
- Наличие собственного ЦОД и облачных компонентов
- Недостаточная сегментация сети и мониторинг
